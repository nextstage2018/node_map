# NodeMap 最終仕上げ実装指示

## 前提

ドキュメント再構成が完了済み。以下の4ファイルが現行のSSOT:

| ファイル | 内容 |
|---|---|
| `CLAUDE.md` | 作業ガイド（ルール10箇条・API・Cron・環境変数） |
| `docs/TABLE_SPECS.md` | DB現状マスタ（全テーブルCREATE文・制約） |
| `docs/FEATURES.md` | 全機能仕様（データフロー・ルール・チェックリスト） |
| `docs/IMPROVEMENT_PLAN.md` | ★今回の実装対象 — 全機能改善計画 |

## 実装対象

`docs/IMPROVEMENT_PLAN.md` に記載された Phase A〜G を順番に実装してください。

### 実装順序

| Phase | 内容 |
|---|---|
| A | 共通ルール（営業時間10:00-19:00平日のみ・カレンダー命名`[NM-Task]``[NM-Job]`・1Ch=1PJ・思考プリセット） |
| B | インボックス（メール休眠化・リアルタイム更新・CW/Slack返信下書き・過去変遷パネル） |
| C | カレンダー（営業時間適用・タスク予定スキップ・複数候補出力） |
| D | 組織・ビジネスログUI統合（階層統合・自動紐づけ・ビジネスログをPJ配下に） |
| E | タスクAI会話改善（思考プリセット注入・外部資料取込動線） |
| F | ナレッジ可視化（今日/今週/今月のノード表示） |
| G | 秘書AI全機能接続（不足intent追加・全操作を秘書から実行可能に） |

### 実装の大原則

1. **全改善項目は「秘書からも同じことができる」前提で作る** — 各Phase実装時に対応するintent/カードアクションを同時作成
2. **1 Phaseずつ実装 → `npm run build` で確認 → commit → 次のPhaseへ**
3. **共通ルールを最初に実装**（Phase A）。これが全機能の土台
4. **CLAUDE.mdの「10のルール」を厳守** — 特にgetServerSupabase()、キャッシュ無効化、params Promise型
5. **変更したテーブルがあれば `docs/TABLE_SPECS.md` も更新**
6. **各Phase完了時に `docs/FEATURES.md` の該当セクションも更新**

### 設計思想（忘れないこと）

- **NodeMapの最重要機能は思考マップ** — ノードが勝手に溜まる「思考の貯蔵箱」
- **コミュニケーションはSlack/CWなど外部で行い、NodeMapは自動同期・自動蓄積**
- **タスク進行だけはNodeMap内で** — AI壁打ち・フェーズ管理・思考ノード生成のため
- **秘書AIは将来UIを秘書チャットのみに集約する** — 秘書ができないことをなくす

### AI応答の思考プリセット（全AIに適用）

思考エンジン: 階層思考（Why×5層）→ 飛び地（横方向連想）→ ストーリー化（課題→心理→意味→価値）
対話スタイル: 壁打ち型。「そもそも」「構造で見ると」
伸二レビュー: 構造理解・課題の深さ・顧客心理・ストーリー性の4観点
カスタマイズ: ユーザーの性格タイプで表現の濃淡を調整

詳細は `docs/IMPROVEMENT_PLAN.md` のセクション0-4を参照。

## 開始

まず `docs/IMPROVEMENT_PLAN.md` を全文読み、Phase A から着手してください。
