作成日: 2026-03-06

## デザイン方針

### 配色（3色ルール）

| 役割 | 色 | 用途 |
|---|---|---|
| **メイン** | Slate（slate-50〜900） | 背景・テキスト・ボーダー全般 |
| **アクセント** | Blue（blue-500/600） | CTA・アクティブ状態・リンク・重要アクション |
| **セマンティック** | 状況に応じた色 | 緑=完了、赤=緊急/エラー、黄=注意（最小限に使用） |

**原則**: 白背景 + slate系テキスト + blue系アクセントを徹底。ページごとに色が変わる現状を統一。

### デザイントーン

- **カード主体**: テキスト羅列をやめ、情報をカード単位で視覚化
- **余白を活かす**: 詰め込みすぎない。呼吸できるレイアウト
- **アイコン+ラベル**: 文字だけの操作ボタンを排除
- **階層が一目でわかる**: パンくず・インデント・グループ化

## サイドメニュー統合（現状11項目 → 6項目）

### Before（現状）

```
秘書
─── コミュニケーション ───
インボックス
─── ワーク ───
タスク / ジョブ / アイデアメモ / ビジネスログ
─── ナレッジ ───
思考マップ / ナレッジ
─── つながり ───
コンタクト / 組織
─── システム ───
設定
```

### After（統合案）

```
秘書                     ← 最上位に維持（メインUI）
インボックス              ← CW/Slackのみ（メール非表示）
タスク                   ← ジョブ・メモも秘書に任せ、タスクだけ残す
思考マップ                ← ナレッジを統合（タブ切替）
組織・プロジェクト         ← 組織・コンタクト・PJ・ビジネスログを階層統合
設定
```

**削除/統合の理由:**

| 現在の項目 | 統合先 | 理由 |
|---|---|---|
| ジョブ | 秘書 | ジョブ承認は秘書カードで完結。専用ページは不要 |
| アイデアメモ | 秘書 | メモ投入→タスク変換は秘書で。閲覧はタスクから |
| ビジネスログ | 組織・PJ内タブ | Phase Dで既にPJ配下に統合済み |
| ナレッジ | 思考マップ内タブ | 閲覧中心。思考マップと同じ文脈 |
| コンタクト | 組織配下 | 組織>メンバー>コンタクトの階層 |

### Phase UI-3: 秘書ページ（最重要）

**目的**: 秘書をNodeMapの「顔」に。エージェントらしい体験を実現

**変更ファイル:**
- `src/components/secretary/SecretaryChat.tsx` — 全面リデザイン
- `src/components/secretary/ChatCards.tsx` — カードデザイン統一・改善
- `src/components/secretary/WelcomeDashboard.tsx` — 新規作成（初期画面）
- `src/components/secretary/QuickActions.tsx` — 新規作成（アクションカード）

**現状の問題:**
- 初期画面がチップの羅列で何をしていいかわからない
- アシスタント応答がテキスト長文で読みづらい
- カードのデザインがバラバラ

**改善:**

#### 3-1. ウェルカム画面（メッセージ未送信時）
```
┌───────────────────────────────────────────┐
│                                           │
│         おはようございます、伸二さん          │
│         今日は3月6日（木）です                │
│                                           │
│  ┌─────────────┐  ┌─────────────┐        │
│  │ 📬 未読 3件   │  │ 📅 予定 2件  │        │
│  │ CW: 2 Slack:1│  │ 10:00 / 14:00│       │
│  └─────────────┘  └─────────────┘        │
│  ┌─────────────┐  ┌─────────────┐        │
│  │ ✅ タスク 5件 │  │ ⚡ ジョブ 2件 │        │
│  │ 進行中: 3    │  │ 承認待ち: 2  │        │
│  └─────────────┘  └─────────────┘        │
│                                           │
│  ── よく使う操作 ──────────────────         │
│  [タスク作成] [日程調整] [下書き確認]        │
│  [プロジェクト確認] [ナレッジ確認]           │
│                                           │
├───────────────────────────────────────────┤
│  💬 何でも聞いてください...          [送信]  │
└───────────────────────────────────────────┘
```

- サマリーカード4枚を2x2グリッドで表示（数値はAPI取得）
- よく使う操作を5つに厳選（チップ形式）
- 挨拶は時間帯で変化（おはようございます / こんにちは / お疲れさまです）

#### 3-2. 会話中の応答デザイン
- **アシスタント応答**: 文章は最大3行まで。それ以上はカードで表現
- **カードデザイン統一**: すべてのカードを以下の構造に統一
  ```
  ┌─ [アイコン] タイトル ──── [アクション] ─┐
  │  コンテンツ（1〜3行）                    │
  │  [ボタン1] [ボタン2]                    │
  └──────────────────────────────────────┘
  ```
- **カード種別ごとの左ボーダー色**:
  - 青: 情報系（ブリーフィング、一覧表示）
  - 緑: 完了系（タスク作成完了、承認完了）
  - 黄: 要対応（ジョブ承認、返信下書き）
  - 赤: 緊急（期限超過、重要メッセージ）

#### 3-3. 入力エリア改善
- テキストエリアを角丸に
- 送信ボタンをアイコンのみ（Send icon）
- ファイル添付アイコンを左配置
- 入力中のサジェストをインラインで表示

#### 7-1. ビジネスログ タイムラインUI

ビジネスログをPJ配下の「タイムライン」タブとして、時間軸で変遷を辿れるUIに強化。

**タイムラインレイアウト:**
```
2026年3月
│
├─ 3/6 ─── [会議] A社キックオフMTG ────────┐
│           参加者: 田中、佐藤               │
│           決定事項: 予算承認、4月開始        │
│           📎 議事録.pdf                    │
│           └──────────────────────────────┘
│
├─ 3/4 ─── [意思決定] 提案書の方向性確定 ───┐
│           背景: 3案から1案に絞り込み        │
│           決定: プランBで進行               │
│           └──────────────────────────────┘
│
├─ 3/1 ─── [イベント] 契約書送付 ──────────┐
│           相手: A社 山田様                  │
│           ステータス: 返送待ち               │
│           └──────────────────────────────┘
│
2026年2月
│
├─ 2/28 ── [メッセージ] CWで見積り相談 ─────┐
│           ...                              │
```

**イベント種別とアイコン:**

| 種別 | アイコン | 色（左ボーダー） | 自動/手動 |
|---|---|---|---|
| 会議 | Calendar | blue | 手動 or カレンダー同期 |
| 意思決定 | GitCommit | purple | 手動 |
| メッセージ | MessageSquare | slate | 自動（CW/Slack同期） |
| タスク完了 | CheckCircle | green | 自動 |
| ファイル共有 | FileText | amber | 自動（Drive同期） |
| マイルストーン | Flag | red | 手動 |
| メモ | StickyNote | slate | 手動 |

**タイムライン機能:**
1. 月ごとのグルーピング（セクションヘッダー）
2. フィルター: 全て / 会議 / 意思決定 / メッセージ / ファイル（チップ切替）
3. 各イベントカードはクリックで詳細展開（アコーディオン）
4. 「+イベント追加」ボタンで手動イベント作成（種別選択→内容入力）
5. 自動蓄積イベント（メッセージ・タスク完了・ファイル）はグレーアウトで区別
6. 秘書から「○○プロジェクトの経緯」で呼び出し可能（既存intent活用）

## 実装順序まとめ

| Phase | 内容 | 規模 | 依存 |
|---|---|---|---|
| **UI-1** | デザイン基盤（配色・コンポーネント） | 中 | なし |

## 注意事項

- **API・サービス層は変更しない** — UIコンポーネントとページのみ変更
- **機能削除ではなく非表示化** — ジョブ・メモのAPIは残す。UIルートだけ削除
- **既存のCard/Badge/Buttonコンポーネントを改善して使い回す** — 新規コンポーネント乱立を避ける
- **秘書のカード種別（28種）はそのまま維持** — デザインだけ統一
- **モバイル対応は今回スコープ外** — デスクトップ優先
