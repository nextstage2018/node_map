## 4. 削除機能の仕様

### 4.1 削除が必要な全エンティティ

| エンティティ | 削除場所 | 確認方法 | カスケード |
|---|---|---|---|
| **組織** | 組織一覧の「…」メニュー | 確認ダイアログ（組織名入力） | プロジェクト → SET NULL、チャネル → CASCADE |
| **プロジェクト** | 組織一覧のPJカード「…」メニュー | 確認ダイアログ | タスク・ジョブ → SET NULL、イベント → SET NULL |
| **テーマ** | タスクタブの「…」メニュー | 確認ダイアログ | マイルストーン.theme_id → SET NULL |
| **マイルストーン** | タスクタブの「…」メニュー | 確認ダイアログ | タスク.milestone_id → SET NULL |
| **タスク** | タスクタブの「…」メニュー | 確認ダイアログ | 会話・ノード → CASCADE |
| **ジョブ** | ジョブタブのカード | ゴミ箱アイコン | カレンダー予定 → 削除 |
| **コンタクト** | 組織詳細メンバータブ | 確認ダイアログ | チャネル → CASCADE |
| **ビジネスイベント** | タイムラインの「…」メニュー | 確認ダイアログ | なし |
| **検討ツリーノード** | ツリーUIのノード右クリック | 確認ダイアログ | 子ノード → CASCADE |
| **ナレッジエントリ** | 設定 > ナレッジ管理 | ゴミ箱アイコン | ノード紐づけ → CASCADE |
| **会議録** | タイムラインまたは検討ツリー | 確認ダイアログ | ツリーノードの source_meeting_id → SET NULL |

### 4.2 削除UIの統一パターン

#### パターン1: 「…」メニュー（組織・PJ・テーマ・MS・タスク・イベント）

```
[···] → ドロップダウン
  ├── ✏️ 編集
  └── 🗑 削除（赤文字）
```

#### パターン2: ゴミ箱アイコン（ジョブ・ナレッジ・メモ）

```
カード右上に Trash2 アイコン（ホバーで赤色）
```

#### パターン3: 確認ダイアログ

重要なエンティティ（組織・PJ）は名前入力で確認：

```
┌──────────────────────────────┐
│  組織「A社」を削除しますか？    │
│                              │
│  この操作は取り消せません。     │
│  関連するプロジェクトとの      │
│  紐づけが解除されます。        │
│                              │
│  確認のため「A社」と入力:      │
│  ┌────────────────────┐      │
│  │                    │      │
│  └────────────────────┘      │
│                              │
│  [キャンセル]  [削除する]     │
└──────────────────────────────┘
```

軽量なエンティティ（タスク・ジョブ等）はシンプル確認：

```
┌──────────────────────────────┐
│  このタスクを削除しますか？    │
│  「競合3社のLP収集」          │
│                              │
│  [キャンセル]  [削除する]     │
└──────────────────────────────┘
```

### 4.3 削除のAPI対応状況

| エンティティ | API EXISTS | 追加が必要 |
|---|---|---|
| 組織 | ✅ DELETE /api/organizations/[id] | UI追加のみ |
| プロジェクト | ✅ DELETE /api/projects/[id] | UI追加のみ |
| テーマ | ❌ | API + UI 新規 |
| マイルストーン | ❌ | API + UI 新規 |
| タスク | ✅ DELETE /api/tasks?id=xxx | UI改善（常時表示） |
| ジョブ | ✅ DELETE /api/jobs?id=xxx | UI移植 |
| コンタクト | ❌ | API + UI 新規 |
| ビジネスイベント | ✅ DELETE /api/business-events/[id] | UI追加のみ |
| 検討ツリーノード | ❌ | API + UI 新規 |
| ナレッジエントリ | ✅ DELETE /api/master/entries?id=xxx | UI改善 |
| 会議録 | ❌ | API + UI 新規 |
