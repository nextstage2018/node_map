### V2 設計概要

**5階層**: Organization > Project > Theme（任意） > Milestone > Task

**タスク vs ジョブ**:
- タスク = 思考を伴う作業。Milestone配下必須。AI評価対象
- ジョブ = 定型業務。プロジェクト紐づけ任意。AI評価対象外

**3つのログ**:
- ビジネスログ = 事実の時系列記録（フルオート・チーム共有）
- 検討ツリー = 会議録からAI生成する意思決定の構造
- 思考ログ = マイルストーン間の個人の思考経路

**会議録が全ての起点**: 検討ツリー生成 + 評価学習 + ビジネスログを同時駆動

**評価エージェントの自己学習**: AI判定 vs 議事録から抽出した人間判定の差分を記録→次回プロンプトに注入

### UI構造（V2）

**サイドメニュー（4項目）**:
- 秘書（/）— 全体俯瞰ダッシュボード
- インボックス（/inbox）— メッセージ
- 組織・PJ（/organizations）— すべての作業の起点
- 設定（/settings）— 個人設定 + ナレッジ閲覧

**プロジェクト詳細（5タブ）**:
- タイムライン / 検討ツリー / 思考マップ / タスク / ジョブ

**削除機能**: 全エンティティに削除ボタンを追加（詳細は UI_V2_SPEC.md セクション4）

### 実装フェーズ

| Phase | 内容 | 依存 |
|---|---|---|
| V2-A | DB: 新規8テーブル作成 + 既存4テーブル変更 | なし |
| V2-B | UI構造変更: サイドメニュー4項目化 + PJ詳細タブ + 削除機能 | V2-A |
| V2-C | テーマ・マイルストーンの CRUD API + UI | V2-A |
| V2-D | 会議録アップロード + AI解析基盤 | V2-A |
| V2-E | 検討ツリー: AI生成 + ツリーUI | V2-D |
| V2-F | チェックポイント: 評価エージェント + 判定UI | V2-C |
| V2-G | 自己学習: 差分記録 + 議事録からの学習 + プロンプト注入 | V2-D, V2-F |
| V2-H | 思考ログ拡張: マイルストーン間スコープ | V2-C |
| V2-I | 統合: 秘書AIへの新intent追加 + ダッシュボード更新 | V2-C〜H |

### 作業ルール

1. 各フェーズは**別スレッド**で実行する
2. 作業スレッド開始時に `CLAUDE.md` と関連ドキュメントを必ず読む
3. ブランチ命名: `feature/v2-a-db-setup` のように `v2-{phase}-{name}`
4. SQL作成後は TABLE_SPECS.md も更新する
5. ビルド確認: `rm -rf .next && npm run build`
6. コミット後にハッシュを報告
7. この管理スレッドに各フェーズの完了報告を記録する

### 注意事項

- VMのディスク容量が逼迫しやすい。`rm -rf .next` を頻繁に実行
- 10のルール（CLAUDE.md冒頭）は必ず遵守
- V2新規テーブルのCREATE文は `docs/ARCHITECTURE_V2.md` セクション7.3にある
- 既存テーブルの変更は `ALTER TABLE ADD COLUMN` で対応（既存データを壊さない）
