# NodeMap — 社内説明資料

## このツールは何か

NodeMapは、普段の仕事の中で生まれる情報を「勝手に」集めて、整理して、活用できる状態にしてくれるビジネスログツールです。

従来の課題として、会議の決定事項がSlackに流れて埋もれる、議事録が属人化する、タスクの進捗が見えない、といった問題がありました。NodeMapはこれらを自動化によって解決します。

皆さんが普段やっていること（Slackで会話する、会議をする、ファイルを共有する）をそのまま続けるだけで、プロジェクトの全記録が自動的に蓄積・整理されていきます。

---

## 大原則：情報は「勝手に」蓄積される

NodeMapの最大の特徴は、手動で何かを登録する必要がほとんどないことです。

### 自動で記録されるもの

| 情報の種類 | どうやって蓄積されるか | あなたがやること |
|---|---|---|
| 会議の内容 | Google MeetのGeminiメモ取り機能 → 自動取り込み → AI解析 | 会議中に「メモを取る」をONにするだけ |
| Slackのやり取り | プロジェクトチャネルのメッセージを定期同期 | いつも通りSlackで会話するだけ |
| Chatworkのやり取り | ルームのメッセージを定期同期 | いつも通りChatworkで会話するだけ |
| タスクの進捗 | ステータス変更時に自動記録 | タスクを完了にするだけ |
| ファイル・資料 | Google Drive連携で自動格納 | いつも通りファイルを共有するだけ |
| 意思決定事項 | 会議録のAI解析で自動抽出 | 会議で「〇〇に決定」と発言するだけ |
| 未確定事項 | 会議録のAI解析で自動検出 | 会議で「検討中」「要確認」と話すだけ |

### ポイント

情報の入口は2つだけです。会議（Gemini会議メモ）と、チャネルメッセージ（Slack / Chatwork）。この2つから、タスク・決定事項・未確定事項・ナレッジがすべて自動生成されます。

「登録し忘れ」が起きないので、プロジェクトの正確な記録が常に残ります。

## 蓄積された情報の可視化

### タイムライン（ビジネスログ）

プロジェクトの出来事を時系列で一覧できる画面です。以下の6種類のイベントが自動的に並びます。

| アイコン | 種別 | 内容 |
|---|---|---|
| カレンダー | 会議 | 会議録が登録されると自動追加 |
| メッセージ | メッセージ | Slack/Chatworkの重要なやり取り |
| チェック | タスク完了 | タスクが完了すると自動記録 |
| ファイル | ファイル共有 | Driveに資料が追加されると記録 |
| フラグ | マイルストーン | マイルストーン達成時に記録 |
| グラフ | 週次サマリー | 毎週月曜に自動生成 |

タイムラインは完全に読み取り専用です。手動で何かを書き込む必要はありません。プロジェクトの「歴史」が勝手にできあがっていきます。

### 検討ツリー（意思決定の可視化）

会議で話し合われたテーマを、ツリー構造で可視化する画面です。

会議録をAIが解析し、3〜7個のテーマに分類。各テーマの下に具体的な論点・決定事項・未確定事項がぶら下がります。複数回の会議を重ねると、同じテーマに新しい情報がマージされていくため、「この件、どこまで話が進んだか」がひと目でわかります。

さらに、AIがタスク提案も自動生成します。「誰が」「何を」「いつまでに」やるべきかが、会議内容から自動的に提案されます。

### 思考マップ（個人の思考経路の可視化）

タスクごとのAI相談の履歴を、Canvas上に可視化する画面です。縦軸が抽象↔具体、横軸が時間（会話ターン）で、どのように思考が深まっていったかを追体験できます。チェックポイント（品質チェック）の位置も表示されます。

## ドキュメントの自動格納

### Driveフォルダ構造

NodeMapは、プロジェクトごとに以下のフォルダ構造をGoogle Drive上に自動作成します。

```
[NodeMap] 組織名/
└── プロジェクト名/
    ├── 定期イベント/       ← 定期MTGの資料
    ├── 会議議事録/         ← 会議メモの格納先
    └── マイルストーン/
        └── MS名/
            └── タスク名/   ← タスク関連の成果物
```

タスクに紐づく関連資料は、このフォルダ構造に自動整理されます。「あのファイルどこだっけ」がなくなります。

### プロジェクトログDoc

各プロジェクトに1つのGoogle Docsドキュメント（プロジェクトログ）が自動作成されます。これがプロジェクトの「正史」です。

会議のたびに、以下が自動追記されます。
- 事前アジェンダ（会議前日にAIが自動生成）
- 会議メモ（Geminiの記録）
- AI解析結果（決定事項・タスク提案・未確定事項）

最新の会議が先頭に来るため、上から読めば常にプロジェクトの最新状況がわかります。

## 1プロジェクト＝1チャンネル

### 基本ルール

NodeMapでは「1つのSlack/Chatworkチャネル＝1つのプロジェクト」として紐づけます。

| チャネル | プロジェクト |
|---|---|
| #proj-クライアントA | クライアントA案件 |
| #proj-社内システム | 社内システム開発 |
| chatwork-room-123 | パートナーB共同PJ |

### なぜ1チャネル＝1プロジェクトか

この紐づけにより、チャネル内の全メッセージが自動的にプロジェクトに紐づきます。Slackで「@NodeMap タスク状況は？」と聞けば、そのチャネルのプロジェクトの情報をボットが即座に返してくれます。

メンバーもチャネルから自動検出されるため、プロジェクトメンバーの登録も不要です。

### ボットでできること

チャネル内で `@NodeMap` にメンションすると、以下の情報を確認できます。

| コマンド | 返ってくる内容 |
|---|---|
| 「タスク状況は？」 | 担当者別のタスク一覧と進捗 |
| 「決定事項は？」 | 直近の意思決定ログ |
| 「未確定事項は？」 | 未解決の論点一覧（社内チャネルのみ） |
| 「アジェンダは？」 | 次回会議の自動生成アジェンダ |
| 「今週のまとめは？」 | 今週の進捗サマリー |
| 「メニュー」 | 選べるコマンド一覧 |

また、Slackでは毎週月曜に今週のブリーフィング、金曜に成果レポートが自動配信されます。

## 会議を起点とした1週間サイクル

NodeMapの運用は、会議を起点とした1週間単位のサイクルで回ります。

### 週次サイクルの流れ

```
【週初め — 会議】
  ① 前週の振り返り
     → 前週のマイルストーンの達成/未達をチェック
     → AIが自動生成した事前アジェンダを確認

  ② 今週やることの議論
     → 「今週末にどうなっていたいか」を言語化

  ③ 会議後（自動）
     → Gemini会議メモが自動取り込み
     → AIが会議内容を解析
     → マイルストーン（今週のゴール）が自動登録
     → タスクが自動提案される
     → Slack/Chatworkに会議サマリーが自動投稿

【平日 — 作業】
  ④ 提案されたタスクを確認・承認
     → Slackのボタンで承認、または検討ツリータブから承認

  ⑤ タスクに取り組む
     → AIに相談しながら進める（壁打ちパートナー）
     → チェックポイントで品質チェック（85点以上で完了可能）

  ⑥ タスク完了
     → 完了通知がSlack/Chatworkに自動送信
     → 関連資料のリンクも一緒に共有
     → マイルストーンの進捗率が自動更新

【週末 — 自動集計】
  ⑦ マイルストーン達成度の判定
     → 全タスク完了 → MS達成
     → 未完了タスクあり → 次週に自動持ち越し

  ⑧ 週次サマリーが自動生成
     → 翌月曜のブリーフィングで配信
```

### 会議前のアジェンダ自動生成

会議の前日夜に、AIが翌日の会議アジェンダを自動生成します。内容は以下の通りです。

- 前回会議からの決定事項の確認
- 未確定事項（まだ結論が出ていない論点）
- マイルストーンの進捗状況（タスク完了率・担当者別）
- 進行中タスクのAI会話要約（何をどこまで考えたか）
- 完了タスクの成果物リンク

このアジェンダはプロジェクトログDocに自動記載され、カレンダーイベントからもリンクで参照できます。会議の準備が大幅に楽になります。

## 日常の使い方まとめ

### あなたがやること

1. 会議ではGoogle Meetの「メモを取る」をONにする
2. Slack / Chatworkでいつも通り仕事の会話をする
3. タスクが提案されたら、Slackのボタンで承認する
4. タスクに取り組み、完了したらステータスを変更する
5. わからないことはタスク画面からAIに相談する

### NodeMapが自動でやってくれること

- 会議内容の記録・解析・共有
- タスクの提案・期限設定・担当者割り当て
- 決定事項・未確定事項の追跡
- 週次アジェンダの自動生成
- 進捗レポートの自動配信
- ドキュメントの自動整理・格納
- タイムラインへの全イベント自動記録

## よくある質問

### Q. 今使っているSlack/Chatworkの使い方を変える必要がありますか？
いいえ。いつも通り使ってください。NodeMapが裏側で自動的に情報を収集・整理します。

### Q. 会議の議事録を自分で書く必要がありますか？
いいえ。Google Meetの「メモを取る」機能をONにするだけで、Gemini AIが自動で記録し、NodeMapが自動で取り込み・解析します。

### Q. タスクは自分で登録する必要がありますか？
基本的にはいいえ。会議内容からAIが自動でタスクを提案します。Slackのボタンで承認するだけです。もちろん手動で追加することもできます。

### Q. 社外のクライアントにも情報が見えてしまいますか？
いいえ。組織の関係性（社内/クライアント/パートナー）に応じて、公開される情報が自動的に制御されます。たとえば社内の未確定事項は、社外チャネルには表示されません。

### Q. 何から始めればいいですか？
まずは担当プロジェクトのSlack/ChatworkチャネルがNodeMapに登録されていることを確認してください。あとは普段通り仕事をするだけで、情報が蓄積されていきます。
