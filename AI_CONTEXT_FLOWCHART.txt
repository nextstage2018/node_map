5. JOB STRUCTURING (4 Branches)
================================================================================

Inbox Message + Job Type Selection
        │
        ├──────┬──────┬──────┬──────┐
        │      │      │      │      │
        ▼      ▼      ▼      ▼      ▼
     schedule consult save_to_drive todo default


BRANCH A: SCHEDULE (日程調整)
──────────────────────────────────────────
Step 1: Get Free Slots
  findFreeSlots(userId, nextWeek_start, nextWeek_end, 60min, 10-19)
  ↓
  Result: [{start, end, durationMinutes}, ...]
  Format: "3/6(木) 10:00-12:30, 14:00-19:00"

Step 2: Get User Metadata
  displayName → user_metadata.name
  emailSignature → user_metadata.email_signature
  writingStyle → getUserWritingStyle(userId, 'email')

Step 3: Claude API (Greeting/Closing Only)
  System: "日程調整アシスタント - 挨拶+締めだけ生成"
  Input: Message content
  Output: JSON {greeting, closing, purpose}
  Note: Dates NOT generated by Claude, inserted by code

Step 4: Hybrid Response Assembly
  response = greeting + "\n\n【候補日時】\n" + freeSlots + "\n\n" + closing
  ↓
  IF email AND signature: append signature
  ↓
  Return: {aiDraft, purpose, freeSlots}


BRANCH B: CONSULT (社内相談)
──────────────────────────────────────────
Step 1: Get Thread Messages
  Find by thread_id OR from_address
  SELECT (last 10) ORDER BY created_at ASC
  ↓
  Result: [{from_name, body, direction, timestamp}, ...]

Step 2: Claude API (Thread Summary)
  System: "スレッド要約アシスタント - 200文字以内"
  Input: Message lines with direction labels
  Output: Plain text summary

Step 3: Return
  {threadSummary, consultQuestion (user input)}


BRANCH C: SAVE_TO_DRIVE
──────────────────────────────────────────
(No Claude API - Pure DB Logic)
Detect organization + project from:
  1. Message metadata (slackChannel / chatworkRoomId)
  2. Email domain (if email)
  ↓
  Return: {projectId, projectName, orgName}


BRANCH D: TODO + DEFAULT
──────────────────────────────────────────
Step 1: Claude API (Generic Job)
  System: "メッセージをジョブ化 - title(20)/desc(50)"
  Input: Message content
  Output: JSON {title, description}

Step 2: Return
  {title, description}
